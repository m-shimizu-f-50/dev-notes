## 上流工程とは何か？

ソフトウェア開発には大きく分けて「上流工程」と「下流工程」があります。

```
【上流工程】 ← あなたが目指したいゾーン
  ┌─────────────────────────────────────┐
  │  要件定義 → 基本設計 → 詳細設計     │
  └─────────────────────────────────────┘
           ↓
【下流工程】
  ┌─────────────────────────────────────┐
  │  実装（コーディング） → テスト      │
  └─────────────────────────────────────┘
```

- **上流工程**：「何を作るか」「どう作るか」を決めるフェーズ
- **下流工程**：決まったことを実装・検証するフェーズ

エンジニアとしてコーディングに慣れてきたら、次のステップとして上流工程を学ぶことでキャリアの幅が大きく広がります。

- 機能要件・非機能要件の整理

## よくある誤解

❌ **「上流工程はコードを書かないから楽」**
→ コーディング以上に考える量が多い。実装への深い理解があってこそ良い設計ができる。

❌ **「設計書は書けばいいだけ」**
→ 設計書は「関係者全員が同じ認識を持つため」のコミュニケーションツール。書き方・内容・精度が非常に重要。

❌ **「要件定義はビジネス側の仕事」**
→ エンジニアが要件定義に参加することで、実現性の高い要件が作れる。
