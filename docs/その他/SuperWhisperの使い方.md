# SuperWhisperの使い方（無料版・基本用途に絞る）

このドキュメントは **無料版（ローカル音声認識）** 前提で、**普通の使い方**だけを最短でまとめます。用途は **議事録作成** と **開発のアイデア/メモ** が中心です。

## 1. できること（無料版の範囲）

- **音声→テキスト化**：マイク音声を文字起こしして、現在の入力カーソル位置に貼り付ける
- **モード（出力スタイル）**：用途に合わせて整形方針を切り替える
- **カスタムモード（プロンプト）**：自分の指示（プロンプト）で整形ルールを作れる（無料枠の範囲で）

> 注意：プラン/機能は将来変更され得るため、本ドキュメントでは「無料版でよく使う範囲」の操作手順と運用に絞って説明します。

## 2. セットアップ（最小）

### インストール

1. 公式サイト（`https://superwhisper.com`）からダウンロード
2. `.dmg` を開き、Applicationsへ移動
3. 起動

### 初回設定（ここだけ押さえる）

- **マイク権限**：macOSの「システム設定」→「プライバシーとセキュリティ」→「マイク」で許可
- **言語**：日本語
- **ショートカット**：録音開始/停止のキーを1つ決める（例：`⌘ + Shift + Space`）

## 3. 基本操作（いつもの使い方）

### 録音して貼り付ける

```
1) 録音開始（ショートカット or メニューバーから開始）
2) ふつうに話す
3) 録音停止
4) 変換されたテキストが、カーソル位置に挿入される（または履歴に残る）
```

### 精度を上げるコツ（簡単なものだけ）

- **最初に結論**：先に要点を言う（整形が安定する）
- **1〜2分で区切る**：長時間ぶっ通しにしない
- **静かな環境 + 口元に近いマイク**：環境音の影響を減らす

## 4. 用途別：議事録（テンプレで勝つ）

### おすすめ運用

- **会議中は「要点」だけ拾う**（全文よりも後で使える）
- **区切りで停止**：議題ごとに録音を止める（後で編集しやすい）
- **出力はMarkdownに寄せる**：そのまま `docs/` や Notion/Slack に貼れる

### カスタムモード（議事録用プロンプト例）

以下を「カスタムモード」のプロンプトに設定します（設定方法は後述）。

```text
あなたは議事録作成アシスタントです。
入力は会議の発話テキストです。以下のMarkdown形式で要点を整理してください。

- 不確かな情報は推測で補完しない
- 固有名詞/数値/期限は可能ならそのまま残す
- 決定事項・TODO・懸念点を分ける

出力形式:
## 概要
- （会議の目的/結論を1〜3行）

## 決定事項
- （箇条書き）

## TODO
- [担当: ] [期限: ] （箇条書き）

## 懸念点/未決事項
- （箇条書き）
```

### 話し方の例（短く）

```text
議題は認証基盤の刷新。結論はOIDC採用。
決定：今週中にPoC。担当はAさん。期限は金曜。
懸念：既存トークンの移行手順が未確定。
```

## 5. 用途別：開発アイデア/メモ（そのままタスク化）

### カスタムモード（アイデアメモ用プロンプト例）

```text
あなたは開発メモ整理アシスタントです。
入力は思いつきのメモです。推測せず、要点を短く構造化してください。

出力形式（Markdown）:
## アイデア
- （1〜3行）

## 具体化するとしたら
- 要件（箇条書き）
- 仕様案（箇条書き）

## 次アクション
- （そのままIssue/タスクにできる粒度で3つまで）
```

### 話し方の例（短く）

```text
ログ調査が遅いので、エラーパターン別にダッシュボードを作りたい。
まずは500系を分類して、直近7日分の傾向を見たい。
```

## 6. プロンプト（カスタムモード）の設定方法と使い方

### 設定方法（手順）

アプリのUIはバージョンで表記が変わることがありますが、流れは同じです。

1. メニューバーの SuperWhisper を開く
2. **Modes**（モード）を開く
3. **Custom**（カスタム）/ **New Mode**（新規）を選ぶ
4. **Name**（名前）を付ける（例：`議事録（要点）` / `開発メモ（タスク化）`）
5. **Prompt / Instructions** にプロンプトを貼り付ける
6. 保存して、そのモードを選択した状態で録音する

### 使い方（運用）

- **プロンプトは短く、出力形式を固定**：迷わせない（Markdown見出しを指定するのがコツ）
- **推測禁止を明記**：議事録で事実が崩れるのを防ぐ
- **TODOの書式を決める**：`[担当][期限]` など（後で検索/整理しやすい）

## 7. よくあるミス（最低限）

- **長すぎる録音**：精度が落ちやすい → 1〜2分で区切る
- **モードを作らずに運用開始**：整形が安定しない → 議事録/メモの2つだけ作る
- **雑音環境で使う**：誤認識が増える → マイク位置と環境を先に整える
